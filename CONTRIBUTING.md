# Contributing to Tilde

## Reporting bugs

Please file bugs in the [GitHub issue tracker](https://github.com/gphalkes/tilde/issues).
Include the Tilde version, your operating system and terminal, and a description of how to
reproduce the problem.

## Suggesting features

Feature suggestions are also welcome in the issue tracker. Describing the use case — not
just the feature — helps with evaluation.

## Asking questions

For questions about usage or building, the mailing list
**tilde-text-editor &lt;at&gt; googlegroups.com** is the best place to ask.

## Submitting patches

Pull requests are welcome. A few guidelines:

- Keep changes focused. One logical change per pull request makes review easier.
- Follow the existing code style: C++17, `snake_case_t` type names, two-space
  indentation.
- If you are fixing a bug, a brief description of the root cause in the commit
  message is helpful.
- Build and test your change before submitting. The `testsuite/` and `unittests/`
  directories contain the existing test infrastructure.

Please note that Tilde is a spare-time project, so review may not be immediate.
